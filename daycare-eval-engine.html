0. 한 줄 철학

실측 결과 기관 급식은 거의 다 훌륭(영양사 + 어린이급식관리지원센터). 그래서 점수는 의도적으로 거의 만점·공동 상위권으로 보여 어떤 기관도 깎이지 않게(B2B 안전) 하고, 변별은 '우리 원의 빛나는 강점' 한 줄(약점은 절대 미노출)이 맡는다. 그리고 부모가 바꿀 수 있는 집에서 먹는 나머지 끼니로 전환 → 코칭 가치. = 점수 후킹 → 기관 안심 → 집 끼니 전환.

2. 평가 기준 — 7축 (룰베이스·결정론·클라이언트)

임의 점수가 아니라 보건복지부·WHO·NOVA·농진청 기준. 식감 단계(HabEat)는 모호·기관유형차로 2026-06-22 제외 → 8축→7축.

3. 기관 랭킹 시스템

점수기(줄세우기용)

computeDiversityScore (결정론·daycareMode: 과일·유제품 제외, 가공/반복 패널티 ON). 실측 113곳(서울 7개 구) 중앙값 91·평균 90 = 변별력 낮음(다 훌륭) → 화면은 인위적 커브·등급밴드 없이 객관 점수 + 전국 {유형} 상위 X%만(절대값·등급·동네 등수 미노출).

변별 = 8개 강점지표 (코호트 percentile 1위 1개만 긍정 노출)

규칙: percentile ≥60일 때만, 그 원의 최고 1개만. 동률 타이브레이크 = 생선→제철→콩류→통곡물→단백질→채소→국탕→저가공. 약점·저점은 절대 미노출.

⚠️ 불변선: 어떤 기관도 '못해 보이게' 하지 않는다. 부모가 평가받는 느낌·죄책감 0. 순위는 '잘잘못 아님 = 종이 한 장'. 집 끼니 전환은 부담 아닌 '하루의 마지막 퍼즐'.

4. 설계 · 구현 (데이터 흐름)

식단표 사진/PDF→ CLOVA OCR + Claude Sonnet 비전→ 기관명(좌측상단)·월·끼니별 메뉴→ 디렉터리 매칭(구+유형)→ 📦 institution_menus / items / scores→ /api/eval/rank (코호트 percentile)
  • OCR: /api/ocr — CLOVA(표 셀) + Claude Sonnet 비전(빈칸 날짜밀림 보정). 메뉴명·날짜·기관명(스키마 전용필드) 추출. 비전 대체로 DeepSeek-OCR(환각·149초로 부적합)·Qwen3-VL 검토 → Sonnet 유지(Qwen은 정확도 A/B 후 판단). 그 외 텍스트 LLM은 DeepSeek.
  • 기관 매칭: 이미지 좌측 상단 기관명을 institutions 디렉터리(31,819개)에서 매칭 → institution_id+시군구. 폴더/업로드의 구+유형으로 후보를 좁혀 동명이원 방지.
  • 식재료: mapMenuLocal(농진청 표준명·결정론) → 식품군·영양맵. 미매핑은 DeepSeek 폴백.
  • 저장(멱등): institution_menus(institution_id, month UNIQUE = 월 1벌) → institution_menu_items(끼니별 m:n) → institution_scores(점수 + 강점지표 standout_dims jsonb).
  • 멱등 캐시 + 5회 캡: (기관, 월)이 이미 있으면 OCR 생략하고 저장본 결과(~10초 분석 연출). 잘못된 사진 업로드로 스코어 오염·비용 폭주 방지 위해 analysis_count로 (기관,월)당 재채점 최대 5회 캡.
  • 배치 임포터: scripts/import-seoul-menus.ts — 폴더(구/유형/월) 순회 → OCR(동시 4) → 매칭 → 채점 → 적재. PDF는 pdftoppm 래스터.
  • LLM 정책: 텍스트(메뉴 분해·총평)=DeepSeek V4(DeepInfra) · OCR 비전=Claude Sonnet(~₩104/장). 전국 대량 확장 시 Qwen3-VL(~₩4/장·24배↓) 전환 후보 — 날짜매핑 정확도 A/B 통과 시.
