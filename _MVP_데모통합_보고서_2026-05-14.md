## 2. 통합 방식 — 옵션 B 인플레이스 교체

### 2-1. CSS 임베드
`</style>` 직전에 `.fb-mvp-*` prefix CSS 블록 추가 (line 209~263). MVP의 외부 `css/style.css`와 동등하지만 데모 기존 클래스(`.card`·`.btn-o` 등)와 충돌하지 않도록 모두 prefix 처리.

### 2-2. HTML 교체
기존 `s-foodbridge` div의 `.content` 내부를 MVP 4 screen 구조로 교체:
- `fb-mvp-screen-input` (T710 입력 화면)
- `fb-mvp-screen-loading` (T712 로딩 단계 페이드인)
- `fb-mvp-screen-result` (T711 결과 카드 리스트)
- `fb-mvp-screen-error` (T713 에러 핸들러)

데모 상단의 출처 박스(🔬 Fraker + AAP + 식약처)는 그대로 유지.

### 2-3. JS 통합 — `window.fbMvp` IIFE
원본 `/foodbridge/mvp/js/{api,chain,app}.js` (ES module)를 app-demo.html에 임베드하기 위해 모듈 → IIFE로 변환:
- `import`/`export` 제거
- 외부 fetch 경로는 절대 경로 유지 (`/foodbridge/mvp/data/mock_responses.json`, `/foodbridge/mvp/data/food_seed.json`)
- 콘솔 토글 `window.foodbridge.{enableLive,disableLive,mode}` 동일하게 노출
- 4축 라벨·자료 발화 4룰 검증·알러지 회피 안전망 동일

### 2-4. 기존 라우팅 호환
홈 카드 라우팅과 ICFQ 결과 화면에서 `go('s-foodbridge')` 호출 → 끝부분의 `setTimeout` 진입 hook이 `p9FC.init()` 호출. 통합 후에도 동일 코드 경로 유지를 위해 alias 추가:

```js
window.p9FC = { init: function(){ window.fbMvp.init(); } };
```

### 2-5. 데이터 fetch
- Mock 모드: `data/mock_responses.json` 8 시나리오 (`S1`~`S8`) 그대로 fetch, 입력 유사도 매칭 동일
- Real 모드: `window.__FOODBRIDGE_CONFIG__` 주입 시 Supabase Edge Function 호출 (변경 없음)

## 3. 검증 결과

### 3-1. 파일 정합성
- ✅ `<style>` / `</style>` 정합 (style 태그 짝)
- ✅ `<script>` / `</script>` 정합
- ✅ 라인 수 2,447 (예상 범위 2,500~3,000 이내)
- ✅ 백업 파일 `app-demo.html.backup_2026-05-14` (135,196 bytes 원본 보존)

### 3-2. 격리 검증
- ✅ MVP CSS는 모두 `.fb-mvp-*` prefix — 데모 기존 `.card`·`.btn-o`·`.p9-chip` 등과 클래스 충돌 없음
- ✅ MVP JS는 `window.fbMvp` IIFE — 전역 변수 오염 없음 (모듈 내부 변수 noleak)
- ✅ `window.foodbridge`·`window.FOODBRIDGE_MODE` 글로벌 1개만 노출 (의도된 콘솔 토글용)

### 3-3. 데이터 경로
- ✅ `/foodbridge/mvp/data/mock_responses.json` (440줄, 8 시나리오)
- ✅ `/foodbridge/mvp/data/food_seed.json` (59줄, 시드 30개 + 4축 vector)
- ✅ app-demo.html과 같은 deploy/ 루트 → Vercel 정적 서빙으로 자동 라우팅

### 3-4. 모드 자동 동작 (Mock)
`api.js` `resolveMode()` 로직 그대로 IIFE 내 보존:
1. `?mock=1` URL → `mock` 강제
2. `?live=1` URL → `real` 강제
3. `localStorage.FOODBRIDGE_API_LIVE === 'true'` → `real`
4. `CONFIG.SUPABASE_URL`이 `YOUR-PROJECT` placeholder 그대로면 → `mock` 자동
5. CONFIG 채워졌으나 사용자 토글 없으면 보수적 `mock`

→ 현재 `__FOODBRIDGE_CONFIG__` 미주입 상태 = **Mock 모드 자동** ✅
