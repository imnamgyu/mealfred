1. 구조 — 3층 + 2금지 + 1원칙

층주기역할
거시 — 온보딩 아크1~4주+신뢰→관찰→개입→정착. 이 틀 위에서 커리큘럼 유닛이 배치됨
작전 — 주간 목표 포트폴리오일요일(+콜드스타트)시계열 분석 → 목표 2~3개(커리큘럼 유닛에서 선발) + 우선순위. 기존 단일 lever의 일반화
전술 — 일간 전개매일 크론어제 진료차트 ↔ 활성 목표 대조 → 오늘의 한 걸음 선택(전진/피벗/딥다이브/졸업) → 조립식 편지
  • ❌ 사실 재서술 금지: 관찰 사실(화면 식사 등)은 흐름에서 1회만 서술(보통 목표 도입일). 이후 등장은 코드가 차단(블록 설계상 불가). 예외=안전 인터럽트(ARFID 적신호·질식)만.
  • ❌ 중구난방 금지: 일일 주제 전환은 포트폴리오(2~3개) 안에서만, 전환 사유는 코드 규칙(정체 감지)뿐.
  • ✅ 단계 전개: 모든 편지는 '어제까지의 위치 → 오늘의 다음 걸음' — 같은 위치 재방문 금지, 위치는 DB(진도)로 추적.

2. 커리큘럼 12유닛 통합안 + 정량 체득 기준

21강(코어8+선택8+스팟4+조건부1) → 코어 8 + 통합 선택 4 = 12유닛(이사님 "12개"). 스팟 4(질식·성장기록·적신호·정상화)는 수업이 아닌 단기 안내로 별도 유지. 측정: ◎=식단 기록만으로 / Q=질문 보충 필요.

3. 피드백 루프 — 기록·질문·진도

  • DB: curriculum_progress(child_id·unit_id·status[active/progressing/mastered/relapsed/pivoted]·step(사다리 단)·evidence(신호 카운터 JSONB)·started_at·mastered_at·last_signal_at) + weekly_plans.goals(JSONB 배열 2~3개: unit_id·priority·stop_reason). 매 크론이 기록→신호 추출→evidence 갱신→status 전이.
  • 오늘의 질문 정렬(보라 박스): 질문 선택 1순위 = 활성 유닛의 측정 공백 — 식단 입력만으로 판정 신호가 부족할 때(칩 미기록·Q표시 유닛) 그 유닛의 관측 질문을 던진다(예: 유닛8 활성+태그 표본 부족 → "오늘 식탁에서 어떤 말을 건넸나요?"). 기존 QUESTION_MOVES 로테이션은 2순위로 강등. 답변은 evidence에 적립 — 기록→판정→부족하면 질문으로 보충→진도 갱신→내일 코칭 선택 루프 완성.
  • 전개 규칙(코드): 진전(임계 충족) → 같은 유닛 다음 단(딥다이브) / 정체(신호 변화 0 × 5~7일) → 포트폴리오 내 다른 목표로 피벗(stop_reason 기록) / 전 목표 정체 → plateau 쉬어가기+주간 재진단 / 졸업 → 유지 레지스트리로 이동, 코칭 중단 / 재발 감지 → 재활성(과거 진도에서 재개, 처음부터 아님).

4. 조립식 편지 — 표면의 유한화

  • 블록 스키마: unit × 단계(도입|방법|관찰반응|장애물|전진|칭찬|피벗|졸업) × 변형 3~5 — 사전 작성+적대 검수(멀티에이전트 — 커뮤니티 98·팁 36 파이프라인 재사용). 슬롯: {아이 이름, 사실 카드 인용 1개, 다음 걸음}.
  • 조립 규칙: 진도 상태가 블록을 결정론으로 선택. 같은 블록 3일 내 재사용 금지·관찰 사실 인용은 유닛당 1회 — 코드로 보장(확률 아님). polishKo + (선택) LLM 윤문(내용 불변 계약).
  • 자유 작문 잔존 표면(기존 가드 스택 그대로 적용): 주 1회 일요일 회고/intro(Sonnet)·휴면 복귀·월간 — 신선도 담당, 빈도가 낮아 검증 가능.
  • 효과: 편지당 LLM 콜 14→0~2(크론 사망 원인 소멸), 모델 교체 무풍, "몇 가지 예외 처리하면 되나"가 셀 수 있는 질문이 됨(블록 전수 테스트).
