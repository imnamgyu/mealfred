요약

라이브🧠 뇌 + ✍️ 손 양쪽흡수: progress.ts + curriculumUnits.ts + coachDaily.ts

설문 0개로 meal_logs·growth_logs 기록만 가지고 '편식이 좋아지는 중인지'를 28일 비교 수치로 증명(리텐션)하고, 부모 행동을 12개 커리큘럼 유닛의 사다리(steps)·판정(passWhen)·재발(relapseWhen)·관측 질문(probes)으로 가르치는 진척·커리큘럼 레이어다. progress.ts는 효과측정/BMI급변, curriculumUnits.ts는 12유닛 가드 레지스트리(선언이 곧 명세), coachDaily.ts는 크론이 호출하는 순수 신호 빌더·답변 파서.

📚 읽는 데이터셋
  • meal_logs (log_date·slot·menus·refused·note·environment·place·ate_well·duration_min + autonomy·texture·meal_time)
  • growth_logs (measured_on·또래 BMI 퍼센타일)
  • coach_questions (q_date·answer·context.unitProbe = 질문 답변)
  • child_unit_progress (child_id·unit_id·status·step·evidence·last_signal_at·relapse_count)
  • period_summaries (period_type·period_key·metrics — write-only 롤업)
📚

커리큘럼 — 12유닛 (정의·측정)

커리큘럼의 정의·사다리·측정이 이 모듈(curriculumUnits.ts)에 산다. "선언이 곧 명세" — 유닛마다 사다리(steps)·판정(passWhen)·재발(relapseWhen)·관측 질문(probes)이 한 파일에. 전부 순수 함수·LLM 0콜·날짜 단위 카운트. 이번 주 어떤 유닛을 가르칠지 '선택'은 ① 신호선택기(runWeeklyPlanning·candidateUnits)가 하고, 여기 ⑤는 그 유닛의 내용·진척을 정의·측정한다(선택=①, 정의·측정=⑤).
유닛(한글)사다리 핵심 행동 (step1 → step2 …)
① 압박 내려놓기 pressure-off"한 입만"·재촉 멘트 멈추기(남겨도 담담히) → 환호·박수도 담담하게
② 공복 리듬 hunger-rhythm끼니 30분 전 간식·우유 멈추기 → 식사·간식 2~3시간 간격 고정
③ 식탁 무대 table-stage하루 한 끼 화면 끄고 식탁에서 → 주 5끼+ 같은 자리·같은 시간
④ 새 음식 조금씩 노출 exposure-savings새 음식 격일로 말없이 올리기 → 익숙해지면 티스푼 맛보기 초대
⑤ 배부름 존중 fullness-respect"배불러" 인정하고 종료(완식 강요 멈춤) → 적게 담고 더 달라면 추가
⑥ 부모가 메뉴판 parent-model같은 음식 곁에서 말없이 맛있게(하루 1회) → 가족 저녁 주 5회
⑦ 달콤한 협상 끊기 no-bargain"먹으면 ~줄게" 거래 멈추기 → 디저트 지위 중립화(악마화 금지)
⑧ 식탁의 말 table-talk금지어 3종 끊기("한 입만"·"다 먹어야지"·"안 먹으면…") → 객체 중심 질문("당근이 어땠어?")
⑨ 감각·질감 트랙 sensory-texture질감 한 단계 올리기 → 만지기·탐색(IDDSI 사다리)
⑩ 확장 트랙(음식 다리) food-bridge새 음식 다리 놓기 → 딥·곁들임 — 푸드체이닝의 코칭 유닛
⑪ 자율성·참여 트랙 autonomy-part역할 주기(상 차리기 등) → 스스로 덜어 먹기
⑫ 연계·리듬 트랙 link-rhythm거부한 음식 재시도 → 주간 식단표 점검·연계
유닛 상태기계: not_started → active → progressing → maintenance → mastered (재발 시 relapsed·방향전환 pivoted). 각 step은 passWhen(evidence)로 통과 판정(표본 부족이면 null=보류), holdWeeks 유지 후 승급. 관측 질문(probes)으로 신호 보완("오늘 식탁 분위기는?"·"도전 음식 반응은?"). → 출력 unitEvidence/step이 진도 상태기계(advanceProgress)를 굴리고, 골고루 진척이 홈 카드·E-07 목표 포트폴리오로.
1

데이터 파이프라인 — 가져오기 → 가공

이 모듈이 데이터를 어떻게 가져와서 어떻게 가공하는지 단계별.

1
기록 적재
meal_logs를 ProgressRow/CRow(FactRow 확장: autonomy·texture·meal_time)로 로드. 모든 카운트는 '날짜' 단위로 dedup(행 부풀림 금지 — 적대감사 M6).
▼
2
효과측정 28일 비교
computeProgress가 최근28일 agg vs 직전28일(−55~−28) agg를 계산 → variety(2회+·비거부 식재료 수)·refusalPct·enjoyPct·avgDur 4지표를 improved 플래그와 함께 비교(양쪽 entries≥3일 때만 hasComparison).
▼
3
주간 후보 신호 빌더
buildCandSignals가 7일창에서 envBadPct(집 끼니만)·selfPct·texLow·mtOver30Pct·메모일수(압박/협상/끼니전간식)·간식과다일·신규식재료(28일창)·기관거부 식재료를 집계 → CandidateSignals.
▼
4
유닛 발동 점수화
각 UnitDef.trigger(s)가 CandidateSignals를 받아 0~3 발동 강도를 반환(예: pressureMemoDays≥2 → 2+). E-03 주간 종합(runWeeklyPlanning)이 이 점수로 focus 유닛 2~3개를 선발.
▼
5
신호 추출·사다리 판정
선발 유닛의 extract(rows·answers·today)가 7일창 evidence(예: envTablePct7d·targetExposeDays7d·signalToday)를 산출 → steps[].passWhen(e)로 단계 통과, relapseWhen(e)로 재발을 판정(표본 부족 시 null=보류).
▼
6
답변 블렌딩·롤업
parseProbeAnswers가 coach_questions.context.unitProbe + 칩 정확일치만 ProbeAnswer로 통과시켜('잘 모르겠어요'·자유텍스트 미적립) extract의 표본을 보충(G-04). periodMetrics는 기간 행을 요약해 period_summaries로 upsert(병원 차트).
2

핵심 함수

함수역할
computeProgress(rows, todayStr)최근28일 vs 직전28일 4지표(잘먹는 다양성·거부율·완식률·식사시간) 비교 → ProgressResult{metrics·improved·total·verdict·hasComparison}. 홈 진척 카드(손)로 직행.
periodMetrics(rows) / isoWeekKey·monthKey·quarterKey·halfKey·yearKey기간 행 요약 지표 + 기간 키 생성기. 크론이 period_summaries 롤업 upsert에 사용.
bmiTrend(points)최근 두 측정의 또래 퍼센타일 변화 |Δ|≥15p면 급변 플래그(비알람 톤 note). 홈 BMI 경고 줄.
UNITS: Record<UnitId, UnitDef> + UNIT_IDS + TH12유닛 레지스트리(id·label·lever·minWeek·steps·probes·extract·relapseWhen·trigger) + 임계 상수. 선언이 곧 명세 — 가드 4중 분산 금지.
buildCandSignals(rows, today, attendsDaycare)7일/28일창에서 주간 후보 신호(CandidateSignals)를 rows로 계산. 크론이 runWeeklyPlanning에 주입(E-03 발동 입력).
parseProbeAnswers(qs)coach_questions 답변 → ProbeAnswer[]. context.unitProbe·칩 정확일치만 통과(무지·자유텍스트 미적립 — 거짓 적립 차단).

CoachContext 전달 — 🧠 뇌 + ✍️ 손 양쪽

이 모듈이 채우는 CoachContext 필드(통합 아키텍처 §3 데이터 계약):

progress (computeProgress → 홈 진척 카드)candSignals (buildCandSignals → runWeeklyPlanning 입력)weekly focus goals (UnitDef.trigger → E-07 목표 포트폴리오)unit evidence/step (extract·passWhen → 진도 상태기계 advanceProgress)bmiTrend (홈 급변 경고)period_summaries.metrics (periodMetrics 롤업)
5

가드 — 괴식·환각·안전 차단

  • 표본 부족 시 null 반환(판정 보류 — '기록 없음≠없음', 보류는 질문 정렬이 메움)
  • hasComparison 양쪽 entries≥3 게이트(통계적 환각 차단)
  • 모든 카운트 '날짜' 단위 dedup(행 부풀림 금지 — 적대감사 M6)
  • parseProbeAnswers 칩 정확일치만 적립('잘 모르겠어요'·자유텍스트·미지 칩 미적립 = 거짓 적립 차단)
  • 잘먹는 다양성=2회+ AND 비거부 식재료만(거부한 음식을 '잘먹음'으로 오판 금지)
  • food-bridge/link-rhythm 집 끼니만(place≠daycare) — 기관 급식을 부모 통제로 오귀속 금지
  • bmiTrend 비알람 톤(급변이어도 '전문가와 상의' 권유, 진단 단정 금지)
