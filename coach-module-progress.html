비개발자용 — 이렇게 운영됩니다

이 모듈은 편식 코칭을 '한 번의 조언'이 아니라 '12개 수업으로 짜인 커리큘럼'처럼 굴립니다(curriculumUnits.ts). 마치 학교 통지표처럼, 각 아이가 12개 수업(압박 내려놓기, 공복 리듬, 식탁 무대, 새 음식 노출 등) 중 지금 어디까지 와 있는지를 '시작 전 → 진행 중 → 유지 → 졸업' 같은 단계로 추적합니다. 각 수업에는 1~2단계 사다리(쉬운 행동→다음 행동)와 '이 정도면 통과'라는 판정 기준, 그리고 '다시 나빠졌다'는 재발 기준이 들어 있습니다. 중요한 점은 이 판정에 AI(LLM)를 전혀 쓰지 않는다는 것입니다. 전부 부모가 입력한 식사 기록과 칩(버튼) 답변을 순수 계산으로만 채점합니다.

작동 방식은 이렇습니다. 매일 한 번, 코칭 크론(자동 작업)이 어제까지의 식사 기록을 불러와 각 수업별로 '어제 그 행동을 했는가'를 1(했음)·0(안 했음)·판단보류(기록 없음)로 채점합니다. '기록이 없다'를 '안 했다'로 단정하지 않는 것이 핵심 원칙이라, 표본이 모자라면 함부로 점수를 매기지 않고 보류합니다. 보류가 생기면 코칭 편지 끝에 '어제 식탁 분위기 어땠어요?' 같은 칩 질문을 끼워 부모 답을 받아 표본을 채웁니다. 모든 카운트는 '몇 번'이 아니라 '며칠'로 셉니다(같은 날 여러 끼 기록해도 하루로 묶어 부풀림 방지).

여기에 더해 별도의 '효과 측정자'(progress.ts)가 있습니다. 이건 설문 없이 식사 기록만으로 최근 28일과 직전 28일을 비교해 '잘 먹는 식재료 종 수·거부 비율·완식 비율·식사 시간'이 좋아졌는지를 수치로 보여줍니다. 부모에게는 '편식이 좋아지는 중이에요'처럼 격려로, 회사 입장에선 '우리 코칭이 실제로 효과 있다'는 증거(리텐션·투자 설명)로 쓰입니다. 또 키·몸무게 측정 두 번을 비교해 또래 대비 체중 순위가 15계단 이상 급변하면(살이 빠르게 빠지거나 늘면) 알람이 아닌 부드러운 톤으로 '한 번 살펴보세요'를 띄웁니다.

🥗 영양·편식 정보는 이렇게 처리합니다
결핍 판정: '잘 먹는 식재료'는 28일 안에 2회 이상 나오고 거부 목록에 없는 식재료로 셉니다(1회짜리나 거부한 건 안 침). 새 음식 노출이 필요한지는 '결핍·거부 식품이 있으면(missing>0 또는 refused>0)' 자동으로 노출 수업을 켜는 식으로 판단합니다.

반복노출(노출 적금): 새 음식은 '말없이 식탁에 아주 조금씩' 올리는 걸 1단계로 보고 주 2~3회 노출을 하한, 누적 8회 이상을 다음 단계 기준으로 잡습니다(노출 효과 이론을 수치화). 노출 일수를 평생 누적해 모읍니다(폭주 방지로 최근 60개까지만).

괴식·위험 차단: 질감 수업은 '한 단계 위 질감만, 거부하면 즉시 후퇴'를 규칙으로 둬 무리한 점프를 막습니다. 어린이집에서 거부한 음식을 집에서 다시 시도하는 '연계' 수업은 거부 식재료를 정제(cleanRefusal)해 부모의 거부 메모가 음식으로 오인되는 오염을 막습니다.

강요 금지: '압박 내려놓기·배부름 존중·달콤한 협상 끊기·식탁의 말' 수업이 부모 메모에서 '한 입만/다 먹어/억지로/혼냈' 같은 강요 신호와 '먹으면 ~줄게' 보상 거래를 잡아내, 그게 0이 되어야 통과로 칩니다. 식사 시간 30분 초과율이 50%를 넘으면 완식 강요 신호로 봅니다. 디저트는 악마화하지 말고 끼니와 함께 소량 내라는 중립화 규칙을 둡니다. 같은 수업을 이틀 연속 채근하지 않도록 주당 피벗(방향 전환) 횟수도 1회로 제한합니다.
🧒 운영 예시(아린·5세) — 아린(5세·콩류와 채소가 적게 들어옴·소고기/계란은 잘 먹음·미역국 좋아함)의 경우. [입력] 부모가 7일간 기록한 끼니에서 콩류·채소가 거의 안 보이고, 어린이집 식단표엔 시금치 거부가 찍힘. [처리] 진척 레이어가 '새 음식 조금씩 노출' 유닛을 켜고(missing>0이라 trigger 발동), 부모 메모를 훑어 '한 입만/다 먹어'가 있으면 '압박 내려놓기'도 함께 켬. 매일 어제 기록을 보고 '새 음식을 식탁에 올렸는지'를 1/0/판단보류로 채점하고, 표본이 모자라면 코칭 편지에 '어제 도전 음식 반응은요?' 같은 칩 질문을 끼워 표본을 보충. 어린이집에서 거부한 시금치가 4일 안에 집 메뉴로 다시 등장하면 '연계·리듬' 유닛이 +1로 잡힘. [결과] 6일 넘게 새 노출 신호가 없으면 '가르쳤는데 안 됨(정체)'으로 보고 다른 각도(예: 잘 먹는 미역국에 당근 한 가지 곁들이는 '음식 다리')로 피벗. 한편 효과측정(progress.ts)은 최근 28일 vs 직전 28일을 비교해 '잘 먹는 식재료 종 수'가 늘고 거부율이 줄면 '편식이 좋아지는 중이에요'를 부모에게 보여줌.

커리큘럼 — 12유닛 (정의·측정)

커리큘럼의 정의·사다리·측정이 이 모듈(curriculumUnits.ts)에 산다. "선언이 곧 명세" — 유닛마다 사다리(steps)·판정(passWhen)·재발(relapseWhen)·관측 질문(probes)이 한 파일에. 전부 순수 함수·LLM 0콜·날짜 단위 카운트. 이번 주 어떤 유닛을 가르칠지 '선택'은 ① 신호선택기(runWeeklyPlanning·candidateUnits)가 하고, 여기 ⑤는 그 유닛의 내용·진척을 정의·측정한다(선택=①, 정의·측정=⑤).
유닛(한글)사다리 핵심 행동 (step1 → step2 …)
① 압박 내려놓기 pressure-off"한 입만"·재촉 멘트 멈추기(남겨도 담담히) → 환호·박수도 담담하게
② 공복 리듬 hunger-rhythm끼니 30분 전 간식·우유 멈추기 → 식사·간식 2~3시간 간격 고정
③ 식탁 무대 table-stage하루 한 끼 화면 끄고 식탁에서 → 주 5끼+ 같은 자리·같은 시간
④ 새 음식 조금씩 노출 exposure-savings새 음식 격일로 말없이 올리기 → 익숙해지면 티스푼 맛보기 초대
⑤ 배부름 존중 fullness-respect"배불러" 인정하고 종료(완식 강요 멈춤) → 적게 담고 더 달라면 추가
⑥ 부모가 메뉴판 parent-model같은 음식 곁에서 말없이 맛있게(하루 1회) → 가족 저녁 주 5회
⑦ 달콤한 협상 끊기 no-bargain"먹으면 ~줄게" 거래 멈추기 → 디저트 지위 중립화(악마화 금지)
⑧ 식탁의 말 table-talk금지어 3종 끊기("한 입만"·"다 먹어야지"·"안 먹으면…") → 객체 중심 질문("당근이 어땠어?")
⑨ 감각·질감 트랙 sensory-texture질감 한 단계 올리기 → 만지기·탐색(IDDSI 사다리)
⑩ 확장 트랙(음식 다리) food-bridge새 음식 다리 놓기 → 딥·곁들임 — 푸드체이닝의 코칭 유닛
⑪ 자율성·참여 트랙 autonomy-part역할 주기(상 차리기 등) → 스스로 덜어 먹기

🎓 유닛별 졸업 기준 · 확인 방법

각 유닛은 2단계 사다리를 올라야 졸업(maintenance→mastered). 졸업 기준 = passWhen(코드가 자동 판정), 확인 방법 = 기록탭의 '팁'이 어떻게 질문으로 환산되나(probe). 식사기록만으로 판단되면 확인 방법은 공란(자동).

유닛졸업 기준 (1단계 → 2단계)확인 방법 (기록탭 팁 → 질문)
① 압박 내려놓기압박·재촉 메모 0일(7일) → 부정 환경칩 비율 ≤ 상한"오늘 식탁 분위기는 어땠어요?" / "남기거나 거부할 때 어떠셨어요?"
② 공복 리듬끼니 직전 간식 메모 0일 → 과식간식일 ≤ 상한"저녁 전에 간식·우유를 먹였나요?" / "오늘 간식 흐름은?"
③ 식탁 무대화면 없이 식탁 비율 ≥ 1단계 → ≥ 2단계— (환경칩 식사기록 자동)
④ 새 음식 노출주간 노출일수 ≥ 임계 → 누적 노출일수 ≥ 임계— (노출 카운트 식사기록 자동)
⑤ 배부름 존중완식 강요 메모 0일 → 30분 초과 비율 ≤ 상한"그만 먹겠다 할 때 어떻게 했어요?" / "오늘 양 조절은?"
⑥ 부모가 메뉴판가족 식사일 ≥ 1단계(주N) → ≥ 2단계(주5)"저녁, 아이와 같은 음식을 같이 드셨어요?"
⑦ 협상 끊기거래 메모 0일 → 거래 0 + 중립 디저트 1회+"오늘 디저트·간식은 어떻게 줬어요?"
⑧ 식탁의 말금지어 메모 0일 → 객체 중심 대화 ≥ 70%"오늘 식탁에서 어떤 말을 가장 많이 건넸어요?"
⑨ 감각·질감질감 핑거푸드(2단계) → 일반식(3단계) 도달— (질감칩 식사기록 자동) + 보조 "새 질감 시도?"
⑩ 음식 다리주간 새 음식 ≥ 임계(1·2단계 동일·홀딩↑)— (새 음식 카운트 식사기록 자동)
⑪ 자율성·참여스스로 먹기 비율 ≥ 1단계 → ≥ 2단계(셀프서빙)— (자율성칩 식사기록 자동) + 보조 "작은 역할 줬어요?"
⑫ 연계·리듬거부 음식 재시도 → 주간 식단표 점검·연계"거부했던 음식을 다시 올려봤어요?" / "식단표를 봤어요?"

유닛 발동 점수화
각 UnitDef.trigger(s)가 CandidateSignals를 받아 0~3 발동 강도를 반환(예: pressureMemoDays≥2 → 2+). E-03 주간 종합(runWeeklyPlanning)이 이 점수로 focus 유닛 2~3개를 선발.

신호 추출·사다리 판정

▼

답변 블렌딩·롤업
parseProbeAnswers가 coach_questions.context.unitProbe + 칩 정확일치만 ProbeAnswer로 통과시켜('잘 모르겠어요'·자유텍스트 미적립) extract의 표본을 보충(G-04). periodMetrics는 기간 행을 요약해 period_summaries로 upsert(병원 차트).

가드 — 괴식·환각·안전 차단

  • 표본 부족 시 null 반환(판정 보류 — '기록 없음≠없음', 보류는 질문 정렬이 메움)
  • hasComparison 양쪽 entries≥3 게이트(통계적 환각 차단)
  • 모든 카운트 '날짜' 단위 dedup(행 부풀림 금지 — 적대감사 M6)
  • parseProbeAnswers 칩 정확일치만 적립('잘 모르겠어요'·자유텍스트·미지 칩 미적립 = 거짓 적립 차단)
  • 잘먹는 다양성=2회+ AND 비거부 식재료만(거부한 음식을 '잘먹음'으로 오판 금지)
  • food-bridge/link-rhythm 집 끼니만(place≠daycare) — 기관 급식을 부모 통제로 오귀속 금지
  • bmiTrend 비알람 톤(급변이어도 '전문가와 상의' 권유, 진단 단정 금지)
