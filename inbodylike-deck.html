밀프레드 편식
편식, 데이터로 다루다

영유아 편식을 개인화로 다루는 AI 헬스케어 · 대표 임남규
  • 엄마도 포기한 편식 아이가, 3주 만에 당근을 스스로 집어먹었습니다
  • 인바디가 몸을 숫자로 바꿨다면 — 우리는 아이의 한 끼를 숫자로 바꿉니다
  • 측정 → 진단·처방 → 충족 → 재측정, 아동 영양의 닫힌 헬스케어 루프를 향해
  • 분석(AI 엔진)은 우리가 직접 · 충족(개인화 키트)은 업계1위 협력사 '밀프레드' — 한 몸
멘트: 이 한 장면을 본 순간 저는 이 일을 평생 하기로 했습니다. 인바디가 몸을 숫자로 바꿨듯, 저희는 아이의 끼니를 숫자로 바꾸는 회사 '밀프레드 편식'입니다.
PROBLEM
4명 중 1명이 편식, 그런데 부모는 측정조차 못 한다
흔한데 방치되는 통증 · 짧은 골든타임 · 사라진 결과 데이터
0–2세준비기
만 2–6세🟢 열린 창 · 골든타임
6세 이후🔒 닫힘 · 되돌리기 어려움
  • 영유아 4명 중 1명이 편식 — 드문 일이 아니라 방치되는 통증
  • 부모는 아이가 '무엇을·얼마나' 못 먹는지조차 객관적으로 모른다 (끼니기록 흩어짐)
  • 코칭은 일반론, '실제로 나아졌는지' 보여줄 객관적 결과 데이터가 없다
멘트: 편식은 흔하지만, 진짜 문제는 측정이 안 된다는 겁니다. 부모도 우리 애가 얼마나 못 먹는지 객관적으로 모릅니다. 측정되지 않으니까요.
GAP
'개인화해서 편식을 다뤄주는' 서비스는 비어 있다
기존 대안은 통증의 일부만 건드린다
  • 맘카페 = 파편화된 정보, 우리 아이 기준이 아님
  • 사설 편식치료 = 비싸고 접근성 낮음
  • 간편식 구매 = 음식만 줄 뿐, 편식을 '관리'하지 않음
  • → 진단·코칭·충족을 한 몸으로 '개인화해서 다뤄주는' 서비스는 시장에 없다
멘트: 비교축은 객관 측정·개인화 처방·물리 충족·결과 재측정 네 가지입니다. 기존 대안은 다 한 칸씩만 건드립니다. 네 칸을 한 흐름으로 잇는 곳은 비어 있습니다.
AHA MOMENT
엄마도 포기한 조카가, 3주 만에 당근을 스스로 집었다
가설이 아니라 직접 본 작은 증거에서 시작했다
  • 대표의 유치원생 조카 — 엄마도 포기한 '편식이 매우 심한 아이'
  • 편식 교정 방법론을 직접 적용 → 3주 만에 당근을 스스로 집어먹었다
  • 느낌이 아니라 '방법론'이라 통했다 — 그래서 코드로 옮길 수 있었다
  • 이 한 장면이 '밀프레드 편식'의 출발점
멘트: 이게 출발점입니다. 책상머리 가설이 아니라, 제 눈으로 본 작은 증거였습니다. 방법론이 통한다는 확신이 들자마자 이걸 엔진으로 만들기 시작했습니다.
SOLUTION
앎(앱)만으론 안 끝난다 — 진단·코칭·충족 3축
다음 3장에서 라이브로 도는 엔진을 직접 보여드립니다
  • 진단 = 메뉴→식재료·31종 영양 매핑 엔진 (라이브)
  • 코칭 = 결정론 코칭 플랜 엔진, 계획은 코드·작문만 LLM (환각 0)
  • 충족 = 폐기 없는 개인화 소량 배송 (협력사 '밀프레드' 콜드체인)
  • 편식은 '아는 것'으로 안 끝난다 — 다품종 소량 식재료를 실제로 만나야 한다
멘트: 솔루션은 진단·코칭·충족 3축입니다. 앞 두 축은 제가 직접 짠 라이브 엔진이고, 마지막 충족은 협력사 자산입니다. 지금부터 도식과 실제 앱을 나란히 보여드리겠습니다.
ENGINE ①
매핑 플라이휠 — 메뉴를 식재료·영양으로 (라이브)
급식과 같은 도메인 · 환각 0 · 가운데=우리 설계 엔진 / 우측=실제 앱
📱 실제 앱
  • 메뉴 이름 한 줄 → 식재료 → 31종 KDRI 영양 신호등 자동 환원 · NEIS 약 1만 메뉴 코퍼스 플라이휠 · 환각 0
멘트: 왼쪽이 설계 도식, 오른쪽이 실제 앱입니다. 메뉴 한 줄을 식재료와 31종 영양 신호등으로 풉니다. 학교 급식과 같은 도메인이라 환각이 0입니다. 말이 아니라 지금 돌고 있습니다.
ENGINE ②
결정론 코칭 플랜 — 35가지 방법론을 코드로 (라이브)
단순 LLM 래퍼가 아니다 · 가운데=우리 설계 아키텍처 / 우측=실제 앱
📱 실제 앱
  • 복지부 지침 + 35가지 국제 편식 방법론을 하나의 하네스로 코드화 · '계획은 코드, 작문만 LLM' · 아이 1명당 월 ≈ 1,000원
멘트: 이게 기술 해자입니다. 35가지 방법론을 코드 하네스로 박아두고, LLM은 작문만 합니다. 그래서 환각이 0이고 아이 한 명당 월 1,000원으로 굴립니다. 오른쪽이 그 엔진이 매일 발행하는 코치 편지와 영양 점수입니다.
ENGINE ③

얕은 래퍼가 아니라 식재료 지식 네트워크 · 가운데=우리 설계 / 우측=실제 앱
📱 실제 앱
  • 편식 도감 204종 + 궁합·사촌 거미줄로 '다음 한 입'에 다리를 놓는다 — 코칭 플랜·박스 구성의 근거
멘트: 엔진이 얕은 래퍼가 아니라는 증거입니다. 식재료 204종 도감과, 잘 먹는 것에서 다음 한 입으로 다리를 놓는 궁합 네트워크까지 있습니다. 이 지식이 코칭과 박스 구성의 근거가 됩니다.
FULFILLMENT

  • 폐기 없는 개인화 소량 배송은 가정에서 불가능 → 전문 물류가 꼭 필요
  • 협력사 '밀프레드'(대표 동생): 누적 100만 인분 · 5중보냉 48h 콜드체인 · 100평

엔진이 매주 분석해 구성하는 개인화 키트(7종·소량)
멘트: 편식은 앎으로 안 끝납니다. 새 식재료를 입으로 만나야 합니다. 그런데 다품종 소량을 부모가 사면 다 버려집니다. 폐기 없는 개인화 배송은 가정에서 불가능하고, 그걸 이미 누적 100만 인분으로 굴리는 협력사가 우리와 한 몸입니다.
