1

인증서 통이미지 적발 → 30개 네이티브 요소로 재구성

build_cert.py
  • 기존 이름입력용.pptx = 배경 PNG(슬라이드 꽉 참) + "이름 입력" 텍스트박스 1개 = 제목·문구·날짜·디자인 전부 그림 속. 이사님 "기만" 지적 정확.
  • 소스 추적 → 원본은 HTML(D1_참여인증서_0813.html). HTML→PDF/PNG로 구워 통이미지가 됨.
  • ⭐ HTML 스펙(A4가로·금테·메달·네이비 타이틀·로고)을 python-pptx로 30개 네이티브 요소 재현. 이름칸=진짜 텍스트박스(연회색 "이름" 플레이스홀더). 8/13·8/14 2슬라이드.
원칙 · "이름 넣는 인증서"가 정작 이름칸 빼고 다 그림 = 편집 불가 → 모든 요소를 개별 도형·텍스트로
2

PDF→네이티브 PPTX 범용 변환기 + 트러블슈팅 3종

pdf2pptx.py
  • PyMuPDF로 PDF를 분해: 글자→텍스트박스, 채움/선→도형, 사진→그림(A1 교안·A2 레시피).
  • 로고 검은 막대 → 인라인 이미지 smask(알파) 손실. smask 합성으로 투명 복원. 번호뱃지가 사진에 가림 → 도형을 '큰 배경 / 작은 뱃지' 2패스 분리(작은 건 사진 위).
  • ⭐ 민트 강조박스가 진초록·키워드 증발 → 둘이 아니라 클리핑(진초록 테두리 사각형이 민트 위 클립). 채움 path를 통째 채운 도형으로 + 면적 큰 것부터(뒤) 그리기로 해결.
핵심 · PDF는 클립·알파·z순서가 평면화된 출력물 → 분해 시 하나씩 복원해야 함
3

폰트·렌더러 함정 — Keynote만 검증한 실수 → 미리보기로 재검증

Apple SD Gothic Neo
  • 이사님 캡쳐: A1·A2 글자가 박스 넘쳐 줄바꿈·겹침 = 개판. 원인 ① 맑은 고딕이 맥에 없어 더 넓은 폰트 대체 ② macOS 미리보기가 줄바꿈 끄기 무시.
  • qlmanage로 선생님 렌더러(미리보기) 재현. 폰트를 Apple SD Gothic Neo(맥 기본·Pretendard 자폭 호환→대체 안 일어남)로 교체 + 텍스트박스 넉넉히 확장(투명) → 줄바꿈 무시해도 한 줄 유지.
교훈 · 내가 보는 렌더러(Keynote)와 사용자가 보는 렌더러(미리보기)가 다르면 사용자 환경으로 검증

A2 사진 크롭 — 진짜 이미지 객체 + PowerPoint 네이티브 크롭

crop_top/bottom
  • 이사님: 사진이 카드 틀 밖으로 삐져나와 헤더 침범 = 또 클리핑(object-fit:cover). 처음엔 배경+오버레이로 했으나 "그것도 통이미지" 거부.
  • ⭐ get_drawings(extended)의 clip 사각형을 뽑아 각 사진에 PowerPoint 네이티브 크롭 적용 → 편집·이동·교체 되는 이미지 객체인데 원본처럼 잘림.
  • 검증: 슬라이드당 사진 5개 따로(크롭4)·텍스트 35·도형 63 = 통이미지 아님.

A0 강사용 수업PPT 22장 네이티브 — 가장 어려운 케이스 4문제 동시 해결

pdf2pptx_a0.py
  • 전수조사: 모든 슬라이드에 인라인 배경(추출 불가) + 텍스트가 이름없는 서브셋폰트 — 흰 글자를 검정으로 읽음·띄어쓰기 공백 없음·이모지 두부.
  • ⭐ 배경=가장자리 색 샘플링→단색/그라데이션 도형 재현 · 글자색=원본↔배경 렌더 픽셀 차분으로 진짜 색(흰 타이틀 복원) · 띄어쓰기=글자 간격 측정해 공백 복원 · 이모지(🥪🍞🥛🥩🦀🥚🌱✅)=각각 이미지 요소.
  • 이전 글자겹침·검은두부 전부 해소. 검증: 슬라이드4 = 이미지 7·텍스트 17·도형 26 = 통이미지 아님.
