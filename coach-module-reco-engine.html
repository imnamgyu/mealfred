요약

부분/휴면 포함🧠 뇌 + ✍️ 손 양쪽흡수: coachRecos.ts + coachMaterials.ts + recoEvidence.ts

결핍 식품군에서 '무엇을 추천할지'(식재료·조합·근거)를 코드가 결정론으로 못 박아 LLM에는 '어떻게 쓸지'(문장·톤)만 맡긴다. 추상 조리법 대신 또래가 실제로 가장 자주 먹는 '음식 이름'·검증된 사촌·강한 궁합만 권해 괴식(미역국+당근)과 환각(지어낸 궁합)을 원천 차단한다.

📚 읽는 데이터셋
  • lib/food-graph.json (foodGraph: strongPairsOf 강한 궁합·verifiedCousinsOf 검증 사촌)
  • lib/kit-dish-matrix.json (kitGuide: dishesForIngredient 실측 count 폴백)
  • lib/ingredient-recipes JSON (route가 로드→FreqMap 주입: 또래 급식 음식 빈도)
  • lib/kit-dish-matrix.json (comboMatrix: scoreCombo 조합 검증)
  • lib/ingredient-freq.json (ingredientFreq: topPctOf 급식 상위%)
  • NUTRIENT_FOODS / GROUP_TARGET (nutrition: 영양소 역인덱스·식품군 권장치)
  • coach-tips.json (온보딩 즉효 팁)
1

데이터 파이프라인 — 가져오기 → 가공

이 모듈이 데이터를 어떻게 가져와서 어떻게 가공하는지 단계별.

1
결핍 신호 입력
영양 거울(computeGroupSignals)이 만든 GroupSignal[](group·level·weeklyEst)을 받아 끼니 채널 6군(과일·유제품 간식 제외)만 남기고 심각도순(red 먼저·채소 가점·주간추정 적은 순) 정렬 → deficiencyWindow/buildIngredientPool로 시급 결핍군 산출
▼
2
잘 먹는 식재료 판정(집·P10)
deriveLikedIngredients가 meal_logs를 place=daycare/snack(차려진 것) 제외하고 집(place null/home)에서 서로 다른 2일+ 등장·거부 이력 없는 것만 liked로, 거부는 refused로 분리
▼
3
대표 식재료 회전·랭킹
GROUP_INGREDIENTS에서 결핍군 대표 식재료를 뽑아 rankIngredients(시급도·급식빈도%·강한 궁합·검증 사촌 4기준 가중합)로 정렬, buildIngredientPool은 주간 5개 풀을 mode(supply/challenge/mixed)로 선정
▼
4
3일 무재사용 회전
rotateRecommendation이 recentRecos(최근 3일 추천)에 없는 최상위를 골라 같은 식재료 연속 추천 차단(6/2·6/3 콩 반복 사고 방지), 전부 쿨다운이면 랭킹 1위 폴백
▼
5
조합·궁합·사촌 근거 생성
buildValidatedCombos가 잘 먹는 음식×추천 식재료를 scoreCombo 임계 통과분만 남기고(미역국+당근 탈락), strongPairsOf(강한 궁합)·verifiedCousinsOf(검증 사촌)로 푸드체이닝 라인 구성, 주식 곡물은 stapleDisplay(밥·빵·면)·생채소는 cookedName(찐 단호박) 표기
▼
6
사실 블록 산출→두뇌·손 주입
buildRecoFacts가 '오늘 타깃 음식+또래 인기 음식+곁들임' 멀티라인 text를 만들어 CoachContext.bridgeFacts(손)와 brainCtx.recoCandidates(두뇌)로 동시 전달, evidenceFor는 식재료별 '급식 상위%·영양역할' 결정론 근거 문구 생성

입력 / 출력 예시

📥 live: route가 meal_logs에서 집 빈도순 likedIng·planFor 타깃·일일 회전 recoIng(targetIngredient)·ingredient-recipes freqMap을 buildRecoFacts에 주입

📥 INPUT
{
  "// buildRecoFacts(아린·5세·콩류/비타민A채소 결핍·소고기/계란/감자 잘 먹음)": "live 경로",
  "likedIngredients": ["소고기", "계란", "감자", "두부"],
  "target": "비타민A채소",
  "targetIngredient": "당근",
  "freqMap": {
    "당근": [{ "name": "당근라페", "freq": 9 }, { "name": "소고기당근볶음", "freq": 6 }],
    "단호박": [{ "name": "단호박찜", "freq": 3 }]
  }
}
📤 OUTPUT
{
  "target": "비타민A채소",
  "cousins": ["단호박"],
  "lines": [
    "[오늘 타깃 비타민A채소] 익힌 당근 (또래 인기 음식: 당근라페·소고기당근볶음 · 잘 먹는 감자 곁들이면 좋아요)",
    "소고기 → 궁합 감자·당근",
    "감자 → 사촌 찐 단호박(또래 인기: 단호박찜)"

bridgeFactsrecoCandidates (brainCtx)recoPoolArr/recoMode/recoIng (route 일일 회전 상태)
5

가드 — 괴식·환각·안전 차단

  • 괴식 차단: buildValidatedCombos가 scoreCombo 임계(기본 2) 통과 조합만 — 미역국+당근 등 미통과 제외, 빈 결과 허용(조합 강요 금지)
  • 환각 차단: 궁합은 strongPairsOf(강한 신호만), 사촌은 verifiedCousinsOf(검증된 것만), 음식은 freqMap/kit-matrix 실측만 — 편지는 이 목록 밖을 지어내지 않음
  • 안전 조리 표기: 생채소·생선·말린콩·알류엔 COOK_ADJ 접두(찐 단호박·삶은 달걀)로 생물 오인 방지, 매운 음식 isSpicyDish 제외
  • 주식 곡물 가드: STAPLE_FORMS로 밀·쌀은 날것 노출 금지 → 빵·면·떡·밥 먹는 형태로만(stapleDisplay)
  • 정직성(evidenceFor): 급식빈도 0(단호박)이면 '상위 N%' 절 생략(0회를 상위 100%로 위장 금지), 영양·빈도 둘 다 없으면 빈 text
  • 선호 신호 정제(deriveLikedIngredients): 급식·간식(차려진 것) 제외·집 2일+·거부 우선 — 당근 급식 오판 차단
  • 온보딩 가드(materialsForLowData): 기록<3일이면 결핍 분석 끄고(환각 위험) 입력 안내+즉효 팁
  • 환각 방지(deficiencyWindow/weeklyExposureTarget): 끼니 채널이 전부 green이면 null 반환
