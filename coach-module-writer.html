요약

라이브✍️ 손(작문)으로흡수: coach.ts + coachChronic.ts

'계획=코드·작문=LLM' 분리 원칙에서 '손' 역할. 두뇌(시나리오 선택)와 데이터(타깃·추천·시계열)가 확정한 입력을 받아, SYSTEM_COACH(편식 방법론 9개 + P1~P10 규칙)에 따라 DeepSeek/Claude로 따뜻한 부모용 편지·오늘의질문을 작문하고, 발행 전 결정론 안전가드·어휘 유사도·LLM 의미검증자로 괴식/환각/반복을 걸러낸다.

📚 읽는 데이터셋
  • coach-tips.json (휴면/저데이터용 편식 팁 풀)
  • CHRONIC_GUIDES (코드 상수 6질환 식이 방향)
  • MOVE_MENU / MOVE_KEYS / SCEN_MOVES / SCEN_OPEN / SCEN_ACT (코드 상수 무브·도입·행동 메뉴)
  • ICFQ_ITEMS / QUESTION_MOVES (코드 상수 질문 풀)
  • coachScenarios (selectScenario·SCENARIOS import)
  • llmCost (UsageRec 타입)
1

데이터 파이프라인 — 가져오기 → 가공

이 모듈이 데이터를 어떻게 가져와서 어떻게 가공하는지 단계별.

1
입력 수신
크론/온디맨드가 조립한 LetterInput(base)을 composeLetter가 받음 — reds/covered/missing(우리 분석값)·favoriteFreq(집 끼니 빈도)·timeseries(시계열 사실)·homeRefused/daycareRefused·factCards(말해도 되는 사실)·weeklyArc·chronicGuidance·bridgeFacts(검증된 추천)·snackEval. 데이터를 *직접 조회하지 않고* 받아서 가공만 한다.
▼
2
전환사실 누수 차단·정규화
최종 시나리오 기준으로 ALLOW_TRANSITION 아닌 시나리오는 '거부→수용 전환' 시계열 제거(sanitizeTimeseries), 거부값 엄격정제(sanitizeRefusals=NOISE 메모 통째 드롭). 만성질환은 chronicGuidanceText로 식이 '방향' 1구절 변환(변비+장트러블 섬유충돌 가드).
▼
3
프롬프트 조립
buildLetterUser가 ctx + history(과거편지 중복금지) + factsBlock + mirrorBlock(영양거울 강제 1구절) + scenario/planTarget/open/action/arc/chronic/snack/move/safety 블록을 합쳐 USER 프롬프트 생성. SYSTEM은 캐싱 대상 SYSTEM_COACH 고정.
▼
4
LLM 작문
callClaude로 DeepSeek(DEEPINFRA/OPENROUTER 키 있으면, 뇌=v4-pro·손=v4-flash) 우선, 실패 시 서킷브레이커로 Claude Haiku 폴백. intro(주 첫 진단)만 Sonnet 승격. 최대 5회 재시도(깨진 JSON 회복). generateLetter가 polishKo(어법) + proofreadLetter(LLM 퇴고) 1회.
▼
5
결정론 안전 재생성
letterDeterministicBad로 환각시점·증상단정·섞기누수·김치괴식·튀김권유·과일↔끼니페어링·해조류↔생선혼동·날곡물 검출 시 최대 2회 재생성(detForbid 호출자 정규식도).
▼
6
유사도 + 의미검증 재생성
letterSimilarity(3-gram 자카드, 전체 0.45·도입 0.40 임계)로 표현 반복 시 무브 회전+regenAvoid 재생성. verifyLetter(LLM 검증자 1콜)로 사실왜곡·행동누수·금지표현·진단재서술 검사→위반시 fixNotes 주입 1회 재작성(개선되고 모든 가드 재통과 시만 채택·S1 대칭). deadlineMs 넘기면 발행 우선 강등.
2

핵심 함수

함수역할
composeLetter통합 작성기(크론·온디맨드 공유). precomputed(시나리오·계획·varyOpener) 주입 → 생성 → 결정론/유사도/의미검증 재생성 → letter·oneliner·plan·verify·modelUsed 반환
buildLetterUserLetterInput → 편지 USER 프롬프트 문자열 조립(영양거울·시나리오·타깃·무브·안전·만성·간식 블록). EPIC C byte-동일 회귀테스트용 export
generateLettercallClaude로 편지 생성 후 polishKo+proofreadLetter 퇴고 → {letter, oneliner}
letterDeterministicBad발행 전 결정론 정규식 가드 — 환각시점·증상·섞기누수·김치괴식·튀김·과일끼니페어링·날곡물 위반이면 true(재생성)
verifyLetterLLM 의미 검증자 1콜 — 사실근거·행동1개·금지표현·진단재서술 검사. 장애 시 fail-open(발행 차단 안 함)
callClaudeDeepSeek 우선·Claude 폴백 LLM 호출 래퍼(서킷브레이커·5회 재시도·usage 적재·JSON 파싱)
generateOnboardingLetter기록<3일 신규자용 온보딩 편지(기록 칭찬+가벼운 평가+안내+팁)
generateQuestion오늘의 질문 1개 생성(실제 로그 음식 짚기·결정론 주제 로테이션·1탭 chips)
chronicGuidanceText부모 입력 만성질환 free-text → 식이 방향 1구절(진단·처방 금지·섬유충돌 가드)

가드 — 괴식·환각·안전 차단

  • 괴식: fruitSaltyMix(단과일+짭짤+섞는동사 동시), fruitMealPairing(과일을 끼니에 곁들임 — 과일은 SNACK_CHANNEL 간식 따로), 김치/깍두기에 섞기·칭찬, 짠맛 입맛발달 칭찬, 해조류↔생선·해산물 혼동, 밀·쌀 날곡물 섞기(밀→빵·면·떡/쌀→밥·떡)
  • 환각: FORBID_TIME(지난달·N개월 등 없는 시점), SYMPTOM_RE(입력에 없는 사레·헛구역·뱉음 단정), factCards 밖 사실·횟수 창작 금지, fenceNotes로 부모메모 격리(주관·지시 아님), sanitizeRefusals/cleanRefusal로 메모형 거부 통째 드롭(거짓 전환 차단), 'N가지 식재료' 가짓수 칭찬 차단
  • 안전(영유아): 매운음식·튀김·초가공(돈까스·너겟·소시지·어묵) 권유 차단, 간식 추천 조리표시(반으로잘라·익혀·잘게) 질식가드 보존, 만성질환 진단·처방·정량(mL)·체중언급 금지
  • 행동: NO_FOOD_ACTION_FRAMES(환경·자율성 시나리오) 음식행동 누수 차단, '오늘의 타깃' 단일·행동 1개(P7), verifyLetter 의미검증(행동2개·계획밖 행동·진단재서술)
