요약

라이브🧠 뇌 + ✍️ 손 양쪽흡수: nutrition.ts + snack.ts

care 식사 기록의 식재료를 영양소로 환산해 "며칠 동안 이 영양소를 커버했나"로 신호등(green/yellow/red)을 산출하고(정량 g이 아닌 빈도 평가), 끼니와 분리한 간식 엔진으로 초가공·가당음료·식사간섭·BMI 방향을 따로 감시한다. 신호등의 red 영양소(reds)는 코칭 두뇌(planFor 시나리오)와 추천 엔진의 결핍 앵커가 되고, 간식 평가는 편지에 부드럽게 합쳐진다.

📚 읽는 데이터셋
  • graphSource.getNutrientMap() (농진청 10.4 정밀맵 GEN_NUTRI, JSON)
  • ingredients-light.json의 cat (catOf 빗대기 입력)
  • meal_logs 행(slot·log_date·menus·ingredients·ate_well·meal_time — SnackRow)
  • 내장 상수: NUTRI_MAP·CATEGORY_NUTRI·CATEGORY_GROUP·FOOD_GROUP·KEY_NUTRIENTS·KDRI_NUTRIENTS·KDRI_VAL_BY_AGE·GROUP_TARGET
1

데이터 파이프라인 — 가져오기 → 가공

이 모듈이 데이터를 어떻게 가져와서 어떻게 가공하는지 단계별.

1
식재료→영양소 매핑
nutrientsOf(ing,catOf): ① 농진청 정밀맵 GEN_NUTRI ∪ NUTRI_MAP(둘 다 정확 출처) 우선, ② 둘 다 없으면 풀 카테고리(catOf)의 CATEGORY_NUTRI 빗대기 폴백 — '모름'으로 비우지 않음
▼
2
날짜별 커버일 집계
ingredientsByDay(날짜별 식재료 목록)를 돌며 그날 커버된 영양소를 Set으로 모으고 coverDays[n]++ — '며칠 동안 닿았나'를 센다
▼
3
밴드별 신호등 판정
영양소를 daily(5일+)/frequent(4일+)/occasional(2일+) 밴드로 나눠 KDRI_BAND_NEED 문턱과 비교. d>=need green, d>0 yellow, d==0 red. 데이터 가드: 기록<3일 보류(yellow/reference), 비타민D는 red 비활성
▼
4
식품군 다양성·점수
computeGroupSignals(8군 daily/rotation 목표 주간추정 weeklyEst), computeFoodGroups(covered/missing), computeDiversityScore(다양성 base + 결핍군 게이트캡 − 가공패널티 − 반복패널티). daycareMode면 과일·유제품 제외
▼
5
간식 분리 분석
evaluateSnacks: meal_logs 전체 행에서 SNACK_SLOTS(am_snack/pm_snack/snack)만 추려 worst-of로 ultra/sugary/good 분류, 같은 날 간식 직후 2.5h내 덜 먹은 끼니(ate_well=false)로 간섭성 계산, BMI band→stance(encourage/maintain/reduce), reds로 결핍보완 간식 추천
▼
6
두뇌·손 핸드오프
reds→CoachSignals.reds(두뇌 시나리오)·evaluateSnacks 입력. SnackEval→snackEvalToPrompt→LetterInput.snackEval(편지 한 블록). KDRI/그룹/다양성 신호등→app/page.tsx 홈 UI 렌더
2

핵심 함수

함수역할
nutrientsOf(ing, catOf)식재료 1개→커버 영양소 배열. 농진청∪NUTRI_MAP 우선, 없으면 카테고리 빗대기 폴백
computeSignals(ingredientsByDay, catOf)KEY 15종 영양소 신호등(NutrientSignal[]). cron이 red만 뽑아 reds로 사용
computeKdriSignals(ingredientsByDay, catOf, ageBand)36종 KDRI 신호등(연령대 기준치·빈도 판정·미매핑은 reference). 홈 모달용
computeGroupSignals(ingredientsByDay, catOf)8식품군 다양성 신호등(weeklyEst·green/yellow/red) + proteinOk
computeFoodGroups(allIngredients, catOf)8군 covered/missing 분해
computeDiversityScore(args)영양 점수 단일 진입점: 다양성 base + 게이트캡 − 가공 − 반복 (집/기관 분리)
computeTimeseries(byDate, menuFreq, catOf, today, opts)추세 사실 문장(채소 N일째 없음·반복 메뉴) — LLM 날짜 추정 차단
evaluateSnacks(args)간식 종합 평가→SnackEval|null (초가공·가당·간섭성·stance·좋은간식)
snackEvalToPrompt(ev, rotate)SnackEval→편지용 한 블록 문자열(체중 단어 금지·예시 로테이션)
goodSnackSuggestions(stance, reds, rotate)stance+결핍→좋은 간식 4종(질식 조리표시 보존·중복제거)
isProcessed(name)메뉴명→초가공(ultra)/가공육(cured) 판정 정규식. snack 엔진도 재사용

가드 — 괴식·환각·안전 차단

  • 빈도 평가 명시(정량 g 아님) — '며칠 커버' 기반, 과대 단정 금지
  • 데이터 가드: 기록<3일이면 신호 보류(yellow/reference), 기록<5일이면 red 보류, 비타민D는 빈도 red 비활성(보충제 영양소)
  • 빗대기 안전망: 미매핑 식재료를 '모름'으로 비우지 않고 카테고리 근사 / KDRI 미매핑은 reference(가짜 빨강 방지)
  • KDRI 제외 5종(나트륨·에너지·수분·불소·크롬) — 결핍 걱정 없는 항목 신호화 안 함
  • 환각 차단: computeTimeseries가 날짜 사실만 계산해 LLM 추정 방지, assertNoVeg는 catOf 신뢰 시에만 '채소 없음' 단정(P4)
  • 질식 안전: 간식 추천에 (반으로 잘라)·(익혀·강판)·(잘게) 조리표시 보존 — coach.ts 프롬프트가 통째 표현으로 못 바꾸게
  • 체중 민감 가드: snackStance/snackEvalToPrompt가 체중·살·다이어트·BMI 단어 금지, 방향(stance)만 노출
  • 가공/반복 패널티 과잉처벌 방지: 끼니 3건 미만이면 미적용, 상한(가공22·반복12), 흰쌀밥·김치·국 등 주식 제외
  • daycareMode: 점심 급식만 평가하는 기관이 과일·유제품 결핍으로 부당 감점되지 않게 제외
