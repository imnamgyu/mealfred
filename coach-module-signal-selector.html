🗺️ 이 문서는 통합 아키텍처 지도의 모듈 ①를 코드 단위로 펼친 명세입니다 — 데이터 파이프라인·입출력 예시·가드.
🧠 이 모듈이 '뇌'입니다. 오늘의 시나리오(각도) 선택 + 음식 추천 검수(일간) 에 더해, ⭐주간 목표 포트폴리오(커리큘럼 유닛) 선택도 여기서 합니다 — 일요일 runWeeklyPlanning의 candidateUnits(신호점수+재발우선·온보딩 게이트)가 이번 주 가르칠 유닛 N개를 고르고 weekly.goals로 잠급니다. 유닛의 정의·사다리·진척 측정은 ⑤ 진척·커리큘럼 레이어가 담당(선택=①, 정의·측정=⑤). 모델: 뇌 = DeepSeek V4-Pro(BRAIN_MODEL=sonnet 역할·실패 시 Claude 폴백).
0

요약

라이브🧠 뇌(판단)로흡수: coachScenarios.ts + coachBrain.ts + coachWeekly.ts

편지가 매일 똑같아지지 않게, 데이터 신호로 '오늘의 코칭 시나리오' 1개를 고르는 두뇌. 일간 전술층은 DeepSeek V4-Pro 두뇌(pickActionByBrain)가 13개 후보 시나리오·최근 3주 주간계획·최근 편지·영양/시계열을 종합해 scenarioId+useFood(음식 다룰지)+approvedRecs(추천 검수)를 고른다. 두뇌는 편지를 쓰지 않는다 — 각도 선택과 추천 취사만 하고 작문·영양평가·추천 본문은 v2 엔진이 한다. 주간 작전층은 일요일 DeepSeek V4-Pro(runWeeklyPlanning)이 다음 한 주 닻(타깃·레버·예산·부모 행동목표)을 잡고, 월~토는 planFromWeekly가 그 닻을 잠가 일관성을 유지한다.

📚 읽는 데이터셋
  • weekly_plans (주간 닻: mission_target·behavior_goal·impression·budget·ledger·goals — 두뇌 weeklyEchoes 입력 및 주간 닻 read/write)
  • coach_letters.context (최근 편지·scenarioId 이력 — recentScenarioIds/pastLetters 입력, 결과 context 저장)
  • SCENARIOS (코드 상수 13개 시나리오 라이브러리)
  • curriculumUnits UNITS/UNIT_IDS (v3 목표 포트폴리오 후보 산출)
  • ingredient-recipes·food-graph·kit-matrix (buildRecoFacts 추천 후보 — recoCandidates로 두뇌에 검수 입력)

데이터 파이프라인 — 가져오기 → 가공

이 모듈이 데이터를 어떻게 가져와서 어떻게 가공하는지 단계별.

1
1) 신호 수집(CoachSignals)
cron이 meal_logs를 집/기관으로 분리해 계산한 timeseries·reds·homeReds·missing·homeMissing·homeRefused·daycareRefused·refused·notes·favoriteFoods·icfqRiskCount·envBadPct 등을 CoachSignals 한 묶음으로 받는다
▼
2
2) 후보 시나리오 메뉴 생성
buildBrainContext가 SCENARIOS 13개 각각에 trigger(signals)를 실행해 '발동가능/비발동' 플래그 + recentScenarioIds로 '최근사용' 플래그를 붙여 메뉴 텍스트로 만든다
▼
3
3) 컨텍스트 조립
후보 메뉴 + 영양거울(nutritionMirrorFromInput) + 음식추천 후보(recoCandidates=buildRecoFacts 결과) + 최근 3주 주간계획(weeklyEchoes) + 최근 5~7통 편지를 하나의 프롬프트 문자열로 합친다
▼
4
4) DeepSeek V4-Pro 두뇌 호출+검수
pickActionByBrain이 DeepSeek V4-Pro(BRAIN_MODEL=sonnet 역할·실패 시 Claude 폴백) SYSTEM_BRAIN으로 호출 → scenarioId·planTarget·moveKey·useFood·why를 JSON으로 받고, scenarioId는 SCEN_IDS 화이트리스트로 검증(밖이면 빈값→폴백), approvedRecs는 useFood=true일 때만 후보 부분집합으로 채움(환각 0)
▼
5
5) v2 결정론 폴백/배선
두뇌 scenarioId가 유효하면 planFor(forceScenarioId)로 강제, 실패·빈값이면 결정론 selectScenario(priority+trigger·최근 2일 제외) 폴백. 구조 프레임(환경·자율성·식감)이면 useFood=false로 정합 강제
▼
6
6) 주간 닻 잠금(작전층)
일요일 runWeeklyPlanning(DeepSeek V4-Pro)이 다음 주 닻 산출(후보 화이트리스트로 타깃 환각 차단·ARFID면 push=0), 월~토 planFromWeekly가 mission_target을 잠그고 무브만 회전+채근 캡(주1회)+행동지연 적용, throw 없이 null/cold 폴백으로 발행 보장

핵심 함수

함수역할
selectScenario(s, recentIds)[coachScenarios] 결정론 폴백 셀렉터 — trigger 충족 시나리오를 priority 내림차순 정렬, 최근 2일 사용분 1차 제외, fresh[0]→fired[0]→plateau 폴백으로 CoachScenario 1개 반환
SCENARIOS (상수 13개)[coachScenarios] 진전축하(100)·ARFID관찰(92)·기록공백(84)·식사분위기(72)·보상역효과(70)·자율성다툼(66)·식감거부(60)·새거부(58)·재노출(54)·집기관격차(50)·영양공백(48)·반복메뉴(46)·정체기(30·폴백) — 각 id·label·priority·theoryBasis·promptHint(LLM 각도지시)·avoid·trigger(signals→boolean)
pickActionByBrain(ctx, candidates, model)[coachBrain] DeepSeek V4-Pro 두뇌 호출 → BrainAction 반환. scenarioId 화이트리스트 검증, useFood는 두뇌 boolean 직접 신뢰, approvedRecs=useFood?후보:[] (텍스트 echo 강제 매칭 제거된 수정본)
buildBrainContext(p)[coachBrain] 13개 시나리오 메뉴(발동가능·최근사용 플래그)+영양거울+추천후보+3주 주간계획+최근편지를 JSON 지시 프롬프트로 조립
scenarioFromId / nutritionMirrorFromInput[coachBrain] id→CoachScenario 역참조(폴백 null), LetterInput에서 영양 한 줄 추출(mirror 없으면 covered/missing 요약)
runWeeklyPlanning(i)[coachWeekly] 일요일 DeepSeek V4-Pro 종합 → WeeklySynthesis(mission·mission_target·lever·budget·impression·behaviorGoal·goals). 후보 화이트리스트 강제·ARFID push=0·실패시 coldSynth 폴백
planFromWeekly(p)[coachWeekly] 월~토 닻 실행 — 안전 인터럽트(축하/적신호/공백) 우선, 비-food 레버는 LEVER_SCENARIO 프레임, food는 mission_target 잠금+채근캡(PUSH_MOVES 강등)+아크 단계 회전. null=폴백
candidateUnits / candidateTargets / healAnchor[coachWeekly] v3 유닛 후보 산출(신호점수+재발우선·온보딩 게이트), 타깃 화이트리스트(집부족>전체부족>거부·스톨타깃 후순위), 구닻 behavior_goal/arc 결정론 치유

CoachContext 전달 — 🧠 뇌(판단)로

이 모듈이 채우는 CoachContext 필드(통합 아키텍처 §3 데이터 계약):

scenarioIdscenarioLabelplanbrainweekly
5

가드 — 괴식·환각·안전 차단

  • scenarioId 화이트리스트(SCEN_IDS) 검증 — 목록 밖이면 빈값→v2 결정론 selectScenario 폴백(환각 시나리오 차단)
  • approvedRecs는 후보(food-graph)의 부분집합만·useFood=false면 빈 배열 — 후보 밖 음식·조합 추가 불가(괴식/환각 0)
  • useFood=false면 bridgeFacts='' 로 음식 추천 본문 제외 — 환경 날에 off-target 음식 박기 차단
  • 구조 프레임(환경·자율성·식감)은 강제 useFood=false 정합 — '한 번에 하나' 위반(요구 3개 편지) 차단
  • 다양성 강제 — 최근 2일 동일 시나리오면 다른 각도로(SYSTEM_BRAIN 원칙③ + selectScenario recentIds 1차 제외)
  • 타깃 화이트리스트(candidateTargets/runWeeklyPlanning) — mission_target은 결핍/거부 후보 안에서만(없는 결핍 지어내기 금지)
  • ARFID 적신호(icfqRiskCount≥2) → push=0 선잠금·neophobia-arfid-watch는 진단·'장애' 단어 금지·1회만 안내
  • 안전 제1원칙 — coachWeekly는 throw 안 함(synth는 cold 폴백, planFromWeekly는 null) → 닻 실패해도 발행 보장
  • stalledTarget 3주 진전0 → 후보 맨 뒤로 밀어 같은 음식 3주째 들이밀기 금지
