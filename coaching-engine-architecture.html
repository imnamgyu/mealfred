시스템 맥락 — 엔진은 누구와 대화하나

C4 Lv.1 · System Context

엔진은 스스로 깨지 않습니다. Vercel Cron이 매일 새벽 깨우면, Supabase에서 식사 기록을 읽고, 필요할 때만 Claude에게 글을 맡긴 뒤, 결과를 다시 Supabase에 넣어 둡니다. 부모는 아침에 이미 만들어진 편지를 읽기만 합니다.

★ 핵심 설계 = 비동기 사전 생성. 부모가 앱을 여는 순간 LLM을 호출하지 않습니다 — 새벽에 미리 만들어 둔 편지를 read만 합니다(온디맨드 API는 폴백). 그래서 아침 화면은 즉시 뜨고, LLM 비용·지연은 새벽 배치에 격리됩니다.

두 갈래 — v2(라이브)와 v3(카나리아)

Container Diagram · 파이프라인 대조

같은 입력(끼니 기록 + 주간 닻)을 받아 두 파이프라인이 갈립니다. v2는 글을 LLM이 쓰되 계획을 코드가 못박고 사후에 3중 가드로 검열하고, v3는 아예 LLM을 안 부르고 미리 검수된 블록을 집합 차감으로 조립합니다. v3 어느 단계든 실패하면 throw 없이 v2로 폴백 — 발행은 언제나 보장됩니다.

추천 게이트 — 조합이 통과하는 경로 — comboMatrix.scoreCombo

최종적으로 '잘 먹는 음식 + 결핍 식재료' 조합을 띄울 때, 4단 폴백을 위에서부터 따집니다. 핵심은 borderline LLM 점수(2)는 실제 레시피 동시출현(cells)이 ≥8일 때만 인정한다는 게이트 — 떡+달걀(score 2·cells 4)을 여기서 한 번 더 거릅니다.
