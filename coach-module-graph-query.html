요약

부분/휴면 포함🧠 뇌 + ✍️ 손 양쪽흡수: foodGraph.ts + kitGuide.ts + ingredientFreq.ts + graphSource.ts

추천 네트워크(궁합 pair·사촌 bridge·식판 tray·음식×식재료 적합도·식재료 급식빈도)를 엔진이 안전하게 질의하는 그래프 조회층. 데이터 출처는 graphSource 한 곳으로 격리(JSON 스냅샷↔SQL 무중단 전환), 소비 함수는 약신호·미검증 사촌을 차단해 괴식·환각 없는 곁들임/사촌/인기음식만 돌려준다.

📚 읽는 데이터셋
  • lib/food-graph.json (edges: pair 궁합 · bridge 사촌 · tray NEIS 식판 공통출현)
  • lib/kit-dish-matrix.json (음식×식재료 cells 동시출현 · scores LLM 적합도 0~3 · dishes 메타)
  • lib/ingredient-freq.json (식재료 급식빈도 freq·rank·topPct)
  • lib/nutrient-map.generated.json (농진청 영양맵)
  • public/ingredients-light.json (도감 경량 nm·cat)
  • public/ingredient-recipes.json (또래 급식 빈도 freqMap)
  • SQL(옵트인): ingredients·ingredient_edges·dish_ingredient_stats (warmGraphFromSql로 id→name resolve)
1

데이터 파이프라인 — 가져오기 → 가공

이 모듈이 데이터를 어떻게 가져와서 어떻게 가공하는지 단계별.

1
단일 읽기경로 로드
graphSource.getEdges()/getDishMatrix()가 food-graph.json·kit-dish-matrix.json을 게터 안에서만 lazy 참조(클라 번들 누수 방지). 기본 L2 JSON 스냅샷.
▼
2
SQL warm(옵트인·서버)
크론이 자녀 루프 전 warmGraphFromSql(supabase) 1회 호출 → ingredient_edges⋈ingredients로 id→name resolve, 안전게이트(edges≥100·cells≥10) 통과 시 _edges/_dishMatrix 캐시 교체+무효화 콜백 호출. 빈약/실패면 JSON 유지(safe degrade).
▼
3
인접맵·인덱스 구성
foodGraph.build()가 RawEdge[]를 양방향 Map<식재료,Neighbor[]>로, kitGuide.build()가 scores를 Map<식재료,DishFit[]>로 구성(lazy·warm 시 무효화 후 재구성).
▼
4
강신호 필터링
strongPairsOf=pair 중 grade='strong'(없으면 strength≥PAIR_MIN_STRENGTH=2)만 — 약신호 s=1(떡+달걀 lift0.72) 차단. verifiedCousinsOf=bridge 중 verified만. dishesForIngredient=scores≥minScore(기본2)만.
▼
5
추천 사실 조립
coachRecos.buildRecoFacts가 타깃 식재료의 또래 인기음식+잘 먹는 것과의 강한 궁합, 잘 먹는 식재료의 검증 사촌(+그 사촌 인기음식)을 lines/text로 직조. 전부 테이블 근거(목록 밖 환각 금지).
▼
6
엔진 소비·CoachContext 주입
급식빈도는 recoEvidence(topPctOf)·coachMaterials 랭킹, 궁합/적합도는 comboMatrix·comboGuard 괴식검증으로 흐르고, 최종 text는 cron이 bridgeFacts로 CoachContext에 주입(두뇌 useFood=false면 빈 문자열).
2

핵심 함수

함수역할
neighborsOf(nm)한 식재료의 모든 이웃(궁합 pair·사촌 bridge)을 bridge먼저→grade→strength→count 순으로 정렬 반환. affinity 정적조회 시드.
strongPairsOf(nm)곁들임 추천용 강한 궁합만 — grade='strong'(없으면 strength≥2). 약신호 곁들임(떡+달걀) 일괄 차단. 전 소비자(coachRecos·coachMaterials·comboMatrix·comboGuard) 일원화.
verifiedCousinsOf(nm)푸드체이닝 chain 추천용 검증된 사촌(bridge·verified)만. 미검증 사촌으로 인한 환각 차단.
dishesForIngredient(nm, minScore=2)이 식재료를 넣기 좋은 음식(LLM 적합도≥minScore, 점수순). 도감 상세·코칭 인기음식·comboGuard 점수조회.
freqOf/topPctOf/isCommon(nm)식재료 급식빈도 메타(freq·rank·topPct). 0회(단호박·요거트)는 정직하게 null(상위100% 위장 금지). 4기준 랭킹·근거 문구 단일 진실원.
warmGraphFromSql(db)SQL→이름shape 캐시 교체(L1 warm). 안전게이트·고아엣지 드롭·dishes 메타는 JSON 유지·수치만 교체. 실패 시 JSON degrade.
getEdges/getDishMatrix/getRecipeFreq/getIngredientsLightgraphSource 단일 읽기경로 게터(동기·lazy). 엔진 모듈의 JSON 직접 import 금지 → 전환 시 누락 방지.
3

입력 / 출력 예시

📥 잘 먹는 식재료 목록 + 오늘 타깃 식품군(콩류)·일일 회전된 타깃 식재료(검은콩) + 또래 급식빈도맵. 단위 헬퍼는 식재료 이름 1개를 받는다.

📥 INPUT
// coachRecos.buildRecoFacts (네트워크 헬퍼를 묶어 호출하는 진입점) — 아린(5세) 크론 입력
{
  "likedIngredients": ["소고기", "계란", "감자", "미역", "두부", "쌀"],
  "target": "콩류",
  "targetIngredient": "검은콩",
  "freqMap": {
    "두부": 92,
    "검은콩": 14,
    "고등어": 60
  }
}

// 내부에서 호출되는 단위 헬퍼 예:
// strongPairsOf("검은콩")  → 검은콩의 강한 궁합(grade=strong / strength≥2)
// verifiedCousinsOf("미역") → 미역의 검증된 사촌(bridge·verified)
// dishesForIngredient("검은콩", 2) → 검은콩 넣기 좋은 음식(LLM 점수≥2)
// topPctOf("당근") → 당근 급식 상위 백분위
📤 OUTPUT
// buildRecoFacts 반환 (RecoFacts) — bridgeFacts로 CoachContext에 들어감
{
  "target": "콩류",
  "cousins": ["병아리콩"],
  "lines": [
    "[오늘 타깃 콩류] 삶은 검은콩 (또래 인기 음식: 콩자반·검은콩밥 · 잘 먹는 두부 곁들이면 좋아요)",
    "미역 → 사촌 다시마(또래 인기: 다시마채조림) / 궁합 두부·들기름",
    "감자 → 궁합 계란·당근"
  ],
  "text": "[오늘 타깃 콩류] 삶은 검은콩 (또래 인기 음식: 콩자반·검은콩밥 · 잘 먹는 두부 곁들이면 좋아요)\n미역 → 사촌 다시마(또래 인기: 다시마채조림) / 궁합 두부·들기름\n감자 → 궁합 계란·당근"
}

// 단위 헬퍼 반환 shape 예:
// strongPairsOf("검은콩") → [{ nm:"두부", kind:"pair", strength:3, basis:"같이 쓰는 레시피 N개", grade:"strong" }]
// topPctOf("당근") → 2   (당근 freq184·상위2%)
📤 전달 — RecoFacts.text가 cron에서 bridgeFacts로 CoachContext에 주입(두뇌 useFood=false면 ''). cousins는 어드민·회전 이력, lines는 손(작문)이 목록 밖을 못 지어내게 하는 그라운딩 근거. freq/궁합/적합도는 coachMaterials 랭킹·comboGuard 괴식검증으로도 흐른다.

5

가드 — 괴식·환각·안전 차단

  • 약신호 곁들임 차단: strongPairsOf가 grade='strong'(없으면 strength≥PAIR_MIN_STRENGTH=2)만 통과 — 떡+달걀(lift0.72) 등 s=1 약신호 추천 경로 일괄 차단
  • 미검증 사촌 차단: verifiedCousinsOf가 verified만 — 환각 사촌 푸드체이닝 방지
  • 환각 차단: buildRecoFacts는 popularDishesFor·STAPLE_FORMS 테이블 근거만 직조('목록 밖 음식·조합 금지', 편지가 지어내지 못함)
  • 정직 null: freqOf/topPctOf가 0회 식재료(단호박·요거트)를 '상위100%'로 위장하지 않고 null 반환
  • 주식 곡물 형태: stapleDisplay/STAPLE_FORMS로 날 곡물명(쌀·밀) 대신 먹는 형태(밥·빵·떡)만 노출
  • SQL safe degrade: warmGraphFromSql 안전게이트(edges<100·cells<10)·고아엣지(id 미해소) 드롭, 실패 시 JSON 스냅샷 유지
  • 클라 번들 누수 방지: JSON은 게터 안에서만 참조(tree-shake 유지) — 클라엔 그래프 51KB 미적재
