### 2-2. 분기 결정 로직

키트 21회 진행 데이터에서 6 차원 위험 신호를 0/1로 채점:
- Q1 새 음식 거부 빈도 (≥2 = 1, else 0)
- Q2 식사 시간 30분 초과 (0/1)
- Q3 식이 다양성 — 식재료 카운트 (≥2 = 1, else 0)
- Q4 식감 거부 1주차 vs 3주차 변화 (0/1)
- Q5 식사 중 부정 감정 빈도 (0/1)
- Q6 성장 우려 (BMI 저하 등, 0/1)

총합:
- **0~2**: `NORMAL_PICKY` — 🌿 안심 (그린 #2E7D32)
- **3~4**: `BORDERLINE` — ⚠️ 주의 (옐로 #F57F17)
- **5~6**: `PFD_SUSPECT` — 🚨 위험 (레드 #C62828)

## 4. Real 모드 전환 가이드 (향후)

본 작업은 mock 모드만 동작합니다. 실 데이터 전환 시:

### 4-1. 행동 데이터 fetch

Supabase Edge Function 또는 RPC로 다음 6 차원 집계 데이터를 받습니다:

```js
// 예시 응답 schema
{
  user_id: 'uuid',
  kit_round: 21,
  measured_at: '2026-XX-XX',
  q1_new_food_reject_rate: 0.18,  // 0.0~1.0
  q2_meal_over_30min_rate: 0.42,
  q3_unique_ingredients_count: 15,
  q4_texture_reject_change: -0.12, // 음수면 개선
  q5_negative_emotion_rate: 0.05,
  q6_growth_concern: false,
}
```

### 4-2. 분기 계산

```js
function computeRisk(data){
  const r = (data.q1_new_food_reject_rate > 0.3 ? 1 : 0)
          + (data.q2_meal_over_30min_rate > 0.5 ? 1 : 0)
          + (data.q3_unique_ingredients_count < 12 ? 1 : 0)
          + (data.q4_texture_reject_change >= 0 ? 1 : 0)
          + (data.q5_negative_emotion_rate > 0.4 ? 1 : 0)
          + (data.q6_growth_concern ? 1 : 0);
  if(r <= 2) return 'normal';
  if(r <= 4) return 'borderline';
  return 'pfd';
}
```

### 4-3. fbIcfq.openReal() 본 구현

현재는 toast placeholder. 향후:
1. `await fetch(SUPABASE_URL + '/rest/v1/icfq_aggregate?user_id=...')`
2. `computeRisk(data)` → stage 결정
3. `setStage(stage)` + MOCKS 대신 실 데이터로 막대 그래프 채움

### 4-4. PDF 다운로드 (현재 placeholder)

`handleAction('pdf')` 본 구현:
- jsPDF 또는 서버 사이드 puppeteer
- 21회 행동 그래프 + 6 차원 점수 + ICD-10 R63.3 참고 + 의사 면담용 1 페이지
